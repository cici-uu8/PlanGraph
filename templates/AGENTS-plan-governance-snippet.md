For any task involving plan, roadmap, spec, checklist, task, progress, state, review, closeout, or execution documents:
1. If `docs/plan_registry.md` exists, plan governance is active. Read it first, then proactively maintain it.
2. Do not treat a planning document as active unless the registry or current project policy says it is active.
3. Do not add new substantive body content to closed or superseded plan documents.
4. After creating a new project-level plan document, register it or refresh discovery as appropriate.
5. If a project-level plan clearly replaces another, supersede the older one. If it is clearly finished with no successor, close it.
6. After governance mutations, run lint and refresh derived outputs before declaring the registry healthy.
7. Plan governance applies to canonical project planning documents, not every temporary note, scratch file, or brainstorming artifact.
