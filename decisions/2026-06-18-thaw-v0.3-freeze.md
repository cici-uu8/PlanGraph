# Decision: Thaw v0.3 Freeze For Local Product-Foundation Work

Date: 2026-06-18

## Context

Commit `8689d3a` recorded a conservative release-state decision: freeze the deterministic `v0.3.x` line at `v0.3.2`, keep SQLite out of the published release boundary, and wait for more validation before promoting heavier infrastructure.

After that decision, the author deliberately chose to continue local-only development into SQLite, MCP, and semantic soft edges. This was an author decision, not a request from Claude, another reviewer, or any external party.

## Decision

Thaw the local development freeze for product-foundation experiments while keeping the public release boundary conservative.

SQLite, MCP, and semantic layers may continue in local commits as derived infrastructure. They must not replace `docs/plan_registry.md` as the source of truth, and they must not be described as stable public release behavior until they pass their own tests, smoke checks, and release gates.

## Rationale

PlanGraph is intended to become more than a one-shot governance script. To reach a CodeGraph-like product shape, it needs a stable read substrate, host-facing query interface, and a carefully separated soft-inference layer. The only way to validate those product foundations is to build small, testable slices and run them against real planning repositories.

The earlier freeze was still correct for release discipline. The thaw changes local development order, not the registry truth model or public support promise.

## Boundaries

- Registry remains the source of truth.
- SQLite is a derived cache and may be rebuilt.
- MCP is a read interface over existing CLI/graph/SQLite behavior.
- Semantic edges are explicit soft hints, not registry facts.
- Ordinary query results must not mix in semantic results by default.
- Semantic output should prioritize pairs with no direct registry hard relation and not in the same workstream.
- No GitHub push is part of this decision.

## Consequences

Local development can continue into SQLite, MCP, and semantic layers without pretending the public release is ready. Future release notes must distinguish tested local infrastructure from stable supported surface.
